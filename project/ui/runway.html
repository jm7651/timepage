모델 개발 , 배포 , 모니터링을 하나의 플랫폼으로 통합하여지속적인 개발과 안정적 운영이 가능한 MLOps 플랫폼입니다
Background
모델의 서비스 적용의 어려움

데이터팀은 서비스 개발이 주 업무가 아니기 때문에, 서비스 개발에 능숙한 인력이 부족합니다. 그렇기 때문에 아무리 성능 좋은 모델을 만든다 한들, 서비스가 가능한 수준의 제품으로 만들어내는 일은 어렵습니다.

모델 성능 추적의 어려움

모델의 성능이 떨어짐을 인지하지 못하면, 수요예측에 실패하고 상품이 품절되거나 재고가 쌓이는 상황이 발생합니다.

모델과 데이터 관리의 어려움

반복된 데이터 정제 코드의 수정과 모델 학습 코드의 변경으로 과거의 모델을 재현하고 분석하는 건 어렵습니다.

협업의 어려움

하지만 머신러닝을 운영한다는 것은 위와 같이 실험만큼이나 혹은 그 이상으로 큰 노력이 필요합니다. 운영을 위한 시스템이 마련되지 않으면, 머신러닝은 더 좋은 서비스를 만드는 도구가 아니라 병목이 됩니다

자유도 높은 모델 개발 환경을 제공합니다

탑재된 모델 개발 도구인 링크 (Link) 를 통해 개발 및 운영 모두에서 활용할 수 있는 학습 파이프라안을 구성할 수 있으며, 파이프라인의 수정과 변형이 용이해 최적의 성능을 제공할 수 있습니다

직관적이고 간결한 사용자 인터페이스을 제공합니다

복잡한 인프라 구성을 사용자 친화적 화면과 직관적인 사용자인터페이스로 설정할 수 있으며 , 개발자가 아니더라도 간단한 학습을 통해 머신러닝 모델을 등록하고 배포할 수 있습니다.

자동 재학습을 지원합니다

실시간으로 데이터 흐름과 변화를 감지하여 운영 모델을 자동으로 재학습하여 머신 러닝 어플리케이션의 성능을 유지합니다

유연한 운영 환경을 제공합니다

실험단계에서 개발된 모델을 실제 환경에 동시 배포하여, 운영에 영향을 주지 않고도 새로운 모델을 평가할 수 있습니다

feature
Runway의 실제화면
01
Login

권한을 획득한 사용자는 Runway 플랫폼에 접근 후, 등록된 계정으로 로그인할 수 있습니다. 로그인 화면은 Runway의 로고를 베이스로 디자인을 해서 메인 비쥬얼을 뽑았습니다.

02
project

Project는 Runway 플랫폼에서 리소스 및 운영을 구분하기 위한 논리적인 단위입니다. 사용자는 Project에 접속하여 Runway 플랫폼의 주요 기능을 사용할 수 있습니다. 웹사이트의 홈과 같은 역할과 디자인 요소를 추가하여 직관적으로 카드 디자인에 폴더스타일을 입혀 디자인을 했습니다.

03
Monitoring

모니터링 섹션은 4가지(CPU,Memory,GPU,Disk)를 차트화하여 보여줍니다. 각 시간마다 차트를 빠르게 보여주고 싶어서 버튼형식으로 접근하도록 설계했습니다. 컨텐츠정보를 왼쪽으로 조금 덜 중요한 메타정보를 오른쪽에 배치하였습니다.

04
User

User는 Runway workspace에 등록된 사용자 정보를 확인하고 관리할 수 있는 기능입니다. Runway는 다양한 사용자가 동시에 접근하여 업무를 수행할 수 있도록 권한에 따라 조금씩 다른 UI를 제공합니다.

05
Data source

Project Owner 사용자는 소유한 Project에서 사용할 신규 Data source를 연결할 수 있습니다

Database side-modal

primary 버튼을 눌렀을 시 오른쪽에서 사이드 모달이 나옵니다. Data base는 유동적으로 추가될 수 있도록 디자인 하였습니다.

06
Dataset
Dataset list

Dataset 생성 단계에서는 3가지 유형 (Local file, Data source, Deployment result) 으로 생성해야 합니다.

Dataset detail

사용자는 Runway 플랫폼에서 관리하는 Dataset 목록을 확인하고,각 Dataset의 개요를 확인할 수 있습니다 좌측에는 contents정보를 우측에는 메타정보를 확인하도록 설계했습니다.

07
Pipeline
Pipeline list

Pipeline은 사용자가 작성한 각 코드블럭의 선후 관계를 정의하여 순차적 실행을 지원하는 워크플로우입니다.

Pipeline detail

Link라는 마키나락스의 ML modeling과 연동이 되어서 Link의 Look으로 디자인 하였습니다. 밑의 테이블은 이 파이프라인으로 스케쥴 한 정보를 넣었습니다.

08
Stream

Stream은 실시간으로 생성된 Data를 Model과 연결하여 실시간 Inference 결과를 제공하는 Model serving 방식의 한 종류입니다. 생각보다 많은 컨텐츠 정보를 담아야해서 카드디자인 형식에 아이콘화하여 모달로 내용을 확인하도록 설계했습니다.

Design system
Runway를 가장 잘 표현할 수 있는 컬러와 가독성이 높은 서체를 중심으로 디자인 가이드를 확립했습니다
