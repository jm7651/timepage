Experience Runway

Data Preparation

You can configure a dataset by acquiring original data files from various data sources with just a few clicks.

Model development

Link, an interactive integrated ML development environment, enables you to quickly access Runway's data sets and start analyzing and modeling data.

Model deployment

The model service can be configured by selecting a method that suits the user's environment, and the operational status of the model service can be monitored using an intuitive UI.

Model re-train

By utilizing the pipelines configured in the model development process, you can quickly relearn and even deploy the model without additional work

프로젝트 설명
MLOps / AI 프로덕트 디자인을 하며 어려웠던 점은?
저희가 만들고 있는 Runway는 AI를 활용한 비즈니스 가치 창출을 극대화하기 위해, ML 모델 운영과 ML 라이프 사이클 관리를 쉽고 빠르게 실행하도록 도와주는 MLOps 플랫폼입니다. Runway를 디자인하며 느낀 가장 큰 장벽은 바로 도메인 지식입니다. 디자이너에게 데이터 사이언티스트만큼의 고도의 AI 지식이 요구되지는 않지만, 제품을 만들기 위해서는 일정 수준 이상의 도메인 지식이 필수적입니다. 사용성을 개선하기 위해 Runway를 직적 사용해보려 할 때에도 데이터셋은 어떻게 만드는지, 배포한 모델을 어떤 식으로 모니터링 하는지와 같은 사용 Flow와 MLOps 전반에 대한 이해도가 있어야 합니다. 레퍼런스를 수집하기 위해 경쟁사 리서치를 하려 해도 결국 관련 지식이 있는 누군가가 도움 없이는 타 서비스를 사용하기 조차 어려웠습니다.
도메인 지식이 없어서 생기는 어려움을 극복한 방법
B2B 제품은 사용자와의 거리가 먼 것이 특징이지만, 운이 좋게도 저희가 일하고 있는 Makinarocks에는 Runway로 프로젝트를 담당하고 있는 ML 엔지니어 분들이 많이 계십니다. ML 엔지니어 분들도 도메인 지식이 없는 구성원이 겪는 어려움에 공감했고 해결하기 위해 다양한 방법을 시도했습니다. 사내 ML엔지니어분께서 프로젝트를 시작하는 순간부터, 모델을 배포하고 모니터링하는 과정까지를 설명하는 세션을 주최하기도 하셨고 해당 세션에서 배운 내용을 바탕으로 Runway를 사용해보니 사용성을 개선할 수 있는 부분들이 이전보다 눈에 띄기 시작했습니다.
