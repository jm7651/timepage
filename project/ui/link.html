A better experience for
Data Scientists

데이터 사이언티스트가 친숙하게 사용하는 주피터랩(JupyterLab)의 확장 프로그램으로, 가독성 있는 파이프라인을 만들어 쉽게 공유할 수 있는 AI/ML 모델 개발 도구입니다.

Overview
마키나락스는 왜
Link를 개발했을까?

Link maintains the usability of JupyterLab andminimizes the learning curve of existing users

User research
painpoint 1
‘Cell 간의 연결성을 확인하기 어렵다.’
Jupyter는 탐색적 데이터 분석에 적합한 툴
빠르고 직관적인 Cell 단위 코드 작성/실행, 결과값 확인
painpoint 2
‘반복적 재 실행을 통한 작업 시간 지연된다.’
Jupyter는 Cell 단위로 코드 작성하고
결과값을 얻기 위해 반복적 Cell을 실행
painpoint 3
‘공유가 어렵고 코드 이해도가 떨어진다.’
Jupyter는 사용자의 경험과 노하우에 따라
사용방법과 개발환경이 상이.
painpoint 4
‘입/출력이 다양하고 호환성이 떨어진다.’
Jupyter는 자유도가 높은 툴이기 때문에
사용자가 원하는 스타일대로 자유롭게 개발
Concept
비정형 데이터를 연결하다
Link™는 주피터랩의 사용성을 그대로 유지하여 기존 사용자의 학습시간을 최소화합니다.
코드의 흐름을 쉽게 확인하고, 최적의 모델을 개발할 수 있습니다.
Link는 jupyter의 extension으로 제작되므로 jupyter가 가지는 제품의 consistens 와 look을 헤칠수 없었습니다.
그럼에도 제품으로서 디자인 특성은 가져야 하여서 고민이 많았습니다.
그래서 Link는 그래픽적 요소보다 제품의 사용성 향상에 집중을 했습니다
회사의 첫 제품이었기에 회사의 블루톤을 메인 컬러로 사용하였습니다. 또한 jupyter의 디자인 룩을
엄격따라 따라가려고 했습니다. 자사의 data scientist들과 계속해서 대화해가면서 니즈를 파악하고
사용성 개선을 위해 UI도 수십번씩 수정을 하였습니다.
해결해야할 문제

1.데이터사이언티스트들이 실험해본 코드들(파편적)을 논리구조적으로 나열할 수 있어야 한다
2.이 코드들은 서로 유기적으로 반응해야 한다
3.데이터사이언티스트들이 주로 사용하는 주피터에서 해결해야한다

디자인 역할

제품의 사용성 향상과 프로덕트 디자인 전반적인 방향과 UX관련 개선을 했습니다. 디자인 시스템을 만들고 디자인동료에게 공유했습니다. 초기에는 XD를 사용하였으나 디자인싱크가 많이 맞지않아 직접 코드수정을 했습니다만 코드 Ownership 문제로 비효율적이지만 옆자리 앉아서 하나하나 검수했습니다. 6개월이 지나고 피그마로 바꾸며 디자인싱크가 90% 가까이 맞았습니다만 Jira를 통해 코드레벨로 디테일한 디자인 검수를 하고 있습니다.

문제해결

데이터사이언티스트들의 모델링 속도개선이 30%이상 나아졌다는 내부피드백 및 외부 데이터사이언티스트들의 피드백을 받았습니다.

직면한 과제

시나리오상에서는 코드를 만들고 그것으로 파이프라인을 만드는 디자인설계를 했었으니 링크를 사용하는 사용자들 대다수가 파이프라인(러프하게)을 만들고 그 셀에 코드를 작성하여 시나리오 대응하는 UX개선을 해야합니다.VS code로 확장하게 되면서 VS code에 맞는 디자인을 해야합니다.
