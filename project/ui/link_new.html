해결해야할 문제

1.데이터사이언티스트들이 실험해본 코드들(파편적)을 논리구조적으로 나열할 수 있어야 한다
2.이 코드들은 서로 유기적으로 반응해야 한다
3.데이터사이언티스트들이 주로 사용하는 주피터에서 해결해야한다

디자인 역할

제품의 사용성 향상과 프로덕트 디자인 전반적인 방향과 UX관련 개선을 했습니다. 디자인 시스템을 만들고 디자인동료에게 공유했습니다. 초기에는 XD를 사용하였으나 디자인싱크가 많이 맞지않아 직접 코드수정을 했습니다만 코드 Ownership 문제로 비효율적이지만 옆자리 앉아서 하나하나 검수했습니다. 6개월이 지나고 피그마로 바꾸며 디자인싱크가 90% 가까이 맞았습니다만 Jira를 통해 코드레벨로 디테일한 디자인 검수를 하고 있습니다.

문제해결

데이터사이언티스트들의 모델링 속도개선이 30%이상 나아졌다는 내부피드백 및 외부 데이터사이언티스트들의 피드백을 받았습니다.

직면한 과제

시나리오상에서는 코드를 만들고 그것으로 파이프라인을 만드는 디자인설계를 했었으니 링크를 사용하는 사용자들 대다수가 파이프라인(러프하게)을 만들고 그 셀에 코드를 작성하여 시나리오 대응하는 UX개선을 해야합니다.VS code로 확장하게 되면서 VS code에 맞는 디자인을 해야합니다.

A better Experience
For Data Scientists

데이터 사이언티스트가 친숙하게 사용하는 주피터랩(JupyterLab)의 확장 프로그램으로,
가독성 있는 파이프라인을 만들어 쉽게 공유할 수 있는 AI/ML 모델 개발 도구입니다.

01
Overview
Link를 어떻게 디자인 했는지?
다수의 Data Scientist들이 산업의 다양한 문제를 해결하기 위해 연구 개발에 몰두하는데 도움이 될 수 있도록,코드를 조금이라도 편하게 보고 관리하기 위한 디자인을 하려고 했습니다.
Link는 데이터 사이언티스트가 친숙하게 사용하는 개발 환경인 주피터 랩의 확장프로그램 형태로 개발되었습니다, 왼쪽 패널영역에 파이프라인을, 노트북영역에 파이프라인을 잇는 Code cell을 Sync하도록 디자인을 했습니다.
02
User research
설문조사와 인터뷰를 통해 Jupyter의 Painpoint를 정리했습니다.
(SKT, Socar, MakinaRocks 데이터사이언티스트등을 대상으로 설문조사 했습니다.)
Pain point1
‘Cell 간의 연결성을 확인하기 어렵다.’
Jupyter는 탐색적 데이터 분석에 적합한 툴 즉, 빠르고 직관적인 Cell 단위 코드 작성/실행, 결과값 확인
Pain point2
‘반복적 재 실행을 통한 작업 시간 지연된다.’
Jupyter는 Cell 단위로 코드 작성하고 결과값을 얻기 위해 반복적 Cell을 실행
Pain point3
‘공유가 어렵고 코드 이해도가 떨어진다.’
Jupyter는 사용자의 경험과 노하우에 따라 사용방법과 개발환경이 상이
Pain point4
‘입/출력이 다양하고 호환성이 떨어진다.’
Jupyter는 자유도가 높은 툴이기 때문에 사용자가 원하는 스타일대로 자유롭게 개발.
코드를 쉽고 빠르게 사용할 수 있게 만드는 것이 문제 해결의 핵심입니다.

05
Link remote
파이프라인 원격 실행이 가능해요
자체적인 원격 실행 기능(Execute Pipeline with Remote Servers)을 제공합니다.이 기능을 통하여, 파이프라인 컴포넌트마다 각각의 원격 서버 자원을 할당하여 효율적으로 서버 자원을 이용할 수 있습니다.
06
Link caching
캐싱(Caching) 관리가 가능해요
한 번 실행한 셀의 결과 값을 별도로 관리하고 캐싱(Caching)하여 업무 효율 저하 및 반복 작업을 회피하고 효율을 극대화할 수 있습니다.

파이프라인 컴퍼넌트를 활용하는 다양한 방법이 있어요
Select Header Color
헤더 컬러 설정을 통해 하나 이상의 컴포넌트를 그룹핑 할 수 있습니다. 유사한 기능을 수행하는 컴포넌트를 동일한 컬러로 설정하여 복잡하게 구성된 파이프라인의 구조를 쉽게 파악할 수 있습니다. (파이프라인의 가독성 향상)
Add Comments
파이프라인 컴포넌트에 코멘트를 (컴포넌트에 대한 설명이나 질문) 추가 할 수 있습니다. 저장되거나 공유될 때도 해당 코멘트가 포함됩니다. 코멘트 기능을 활용하면 협업 시 원활한 소통 도구로 활용할 수 있습니다.

실행 완료 된 컴포넌트에 포함 된 요약 정보(Object, Function, Class, Module)를 제공합니다.

실행버튼 리뉴얼
실행버튼이 hover를 했을때 뜨는방식에서 클릭하면 따른 섹션에 나오는 방식으로 바뀌었습니다 이유는 hover시에 씹히는 경우가 많아 두번 클릭을 해야하는 경우도 있고 hover 방식이 자주 쓰는 인터렉션에서는 좋지 않은것 같았습니다.

처음 기획은 파이프라인 컴퍼넌트 아이콘을 오른쪽 끝쪽에 넣어 인터렉션을 하는것이 었는데 실제 사용자들이 왼쪽부분부터 코드를 쓰고 왼쪽 부분쪽 사이드의 사용이크기때문에 왼쪽으로 실행아이콘들을 옮김으로서 사용성 개선을 했습니다. 위치를 옮기면서 아이콘을 더 심플하게 단순화 하였습니다.
